# 精简策略：保留软件商店 + 多语言，先删第一轮目标

## 目标形态
这是一个“专用型 iStoreOS 容器”，不是通用型 iStoreOS。

目标是：
- 在飞牛中以高权限 ARM64 容器运行
- 提供 Web 管理
- 保留软件商店
- 保留多语言
- 提供旁路由能力
- 预装并支持 OpenClash
- 先做温和裁剪，不一开始就动核心依赖

## 当前产品决策

### 保留项
- LuCI
- 软件商店
- 多语言包
- OpenClash
- 旁路由核心能力

### 第一轮删除项
- 多媒体
- 打印
- 蓝牙
- 下载器
- 多余主题

## 裁剪原则
1. **先删最不相关的业务插件**，不先动系统底层。
2. **每删一类，做一次回归验证**。
3. **主题只删多余项，至少保留一个默认主题**。
4. **软件商店和多语言先不动**，避免误伤体验和依赖链。

## 推荐验证顺序

### 第 1 轮：删明显无关项
删：下载器、多媒体、蓝牙、打印、多余主题。
验：
- Web 能打开
- 软件商店页面正常
- OpenClash 页面存在
- 容器能启动

### 第 2 轮：观察依赖面
做：
- 列出残留无关插件
- 看商店和 OpenClash 的实际依赖链
- 再决定是否继续动共享组件

### 第 3 轮：收紧系统级依赖
仅在前两轮稳定后再考虑：
- 无关驱动
- 无关文件系统
- 无关网络协议

## 回归测试清单

### 基础可用性
- 容器启动成功
- `ubus` 正常
- `rpcd` 正常
- `uhttpd` 正常
- LuCI 首页可访问

### 软件商店
- 商店页面可打开
- 应用列表能正常加载
- 基础安装/更新入口不报错

### OpenClash
- 插件页面可打开
- 核心可下载/可识别
- 配置可保存
- 服务可启动
- TUN 模式可工作

### 旁路由
- 独立 IP 可访问
- DNS 可劫持/转发
- 默认网关切换后客户端能上网
- 透明代理/策略路由工作正常
